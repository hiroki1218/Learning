## 1. なぜ vim を学ぶのか

- Linux サーバには GUI がない。SSH でサーバに入って設定ファイルを編集するとき、使えるエディタは基本的に vi/vim のみ
- AWS の EC2 インスタンスでも同様。**インフラを触る人間にとって vim は必須スキル**
- コードを書く時間の大半は「移動・削除・修正」であり、vim はこの無駄を極限まで減らすために設計されている
- ホームポジションから手を動かさずに全操作が完結する → マウスもカーソルキーも不要
- vi はほぼ全ての Unix/Linux 系 OS に最初から入っている

## 2. vim の基本思想と設計原則

### モーダルエディタ
- **モードによって同じキーの意味が変わる**エディタ
- `j` を押すと → ノーマルモードでは「1行下に移動」、入力モードでは「j という文字を入力」
- この仕組みのおかげで、単キー1つだけで移動も編集もできる

### 大文字の法則
- **小文字コマンドを大文字にすると「話が大げさになる」か「動きが逆になる」**
- 例1（大げさ）：`i`（カーソル位置で入力）→ `I`（行頭から入力）
- 例2（逆方向）：`o`（下に空行）→ `O`（上に空行）
- この法則は vim 全体に一貫して適用される

### コマンドの組み合わせ
- vim のキーはプログラムのように組み合わせて使う
- `数字 + コマンド` → その回数分繰り返す（例：`5j` = 5行下へ）
- `操作 + 範囲` → 指定範囲に操作を適用（例：`dw` = 1単語削除）
- この組み合わせが vim の真の力

### 1キータイプの節約
- `$`（行末移動）+ `a`（追加入力）の2回 → `A` の1回で済む
- たった1回のキータイプすら減らす、という思想

## 3. 3つのモード

```
入力モード ←→ 【ノーマルモード】 ←→ コマンドモード
                  （デフォルト）
```

| モード | 役割 | 入り方 | 抜け方 |
|--------|------|--------|--------|
| ノーマルモード | 移動・削除・コピー等の全操作 | vim 起動時のデフォルト | — |
| 入力モード | 文字の入力 | `i`, `a`, `A`, `o` 等 | `Ctrl + C` |
| コマンドモード | 保存・終了・設定変更・検索置換 | `:` | `Ctrl + C` または `Enter` |

- ノーマルモードが中心。常にここを経由する
- 入力モードは「長居する場所ではない」→ 入力が終わったらすぐ戻る

---

## 4. 入力モードへの入り方

| キー | 動作 | 覚え方 |
|------|------|--------|
| `i` | カーソルの**位置**から入力 | **i**nsert |
| `a` | カーソルの**次の文字**から入力 | **a**ppend |
| `I` | **行頭**から入力 | `i` の大げさ版 |
| `A` | **行末**から入力 | `a` の大げさ版 |
| `o` | カーソル行の**下**に空行を作って入力 | **o**pen line |
| `O` | カーソル行の**上**に空行を作って入力 | `o` の逆方向 |

### `i` と `a` の違い

```
  Hello World
  ↑ カーソルが H の上にある場合
  i で入力 → Hの前に文字が入る
  a で入力 → Hの後ろに文字が入る
```

### 実務での使用頻度
- `A`（行末追加）：セミコロン付け忘れ、行末にコメント追加等で**毎日使う**
- `o`（下に新行）：新しいコード行を追加するときに**最も頻繁に使う**入力方法

---

## 5. ノーマルモードに戻る

**`Ctrl + C`** を使う。**`Esc` は使わない。**

- `Esc` はホームポジションから遠すぎる
- vim の売りが「ホームポジションから手を動かさない」なのに `Esc` は矛盾
- `Ctrl + C` は入力モードからもコマンドモードからも使える万能な安全ネット
- Windows の vi でも、古い Solaris の vi でも、全てこれでいける

---

## 6. 保存・終了（コマンドモード）

`:` を押してコマンドモードに入り、以下を入力して `Enter`。

| コマンド | 動作 | 補足 |
|----------|------|------|
| `:w` | 保存 | **w**rite |
| `:q` | 閉じる | **q**uit |
| `:wq` | 保存して閉じる | |
| `:w!` | 強制的に保存 | `!` = 問答無用 |
| `:q!` | 強制的に閉じる（変更を破棄） | **パニック時の生還コマンド** |
| `:wq!` | 強制的に保存して閉じる | |

### コマンドモードの便利機能
- **補完が効く**：`:co` + `Tab` で候補が出て、`Tab` で切り替え可能
- **省略ができる**：`:set tabstop=2` → `:set ts=2`

### ZZ について
- `ZZ`（ノーマルモードで）→ 保存して閉じる（`:wq` と同じ）
- ただし保存したくない場合でも保存してしまうので、癖にするのは非推奨

## 7. 基本移動（hjkl）

```
     k（上）
h（左）   l（右）
     j（下）
```

| キー | 動作 | 覚え方 |
|------|------|--------|
| `h` | 左 | 一番左のキー |
| `j` | 下 | `j` のしっぽが下に伸びている |
| `k` | 上 | `j` の隣（上） |
| `l` | 右 | 一番右のキー |

### もう1つのパターン（Shell / Emacs 共通）

| vim の hjkl | Ctrl 版 | 覚え方 |
|-------------|---------|--------|
| `h`（左） | `Ctrl + b` | **b**ack |
| `j`（下） | `Ctrl + n` | **n**ext |
| `k`（上） | `Ctrl + p` | **p**revious |
| `l`（右） | `Ctrl + f` | **f**orward |

- この `Ctrl` 版は **shell（ターミナル）でも使える**
- shell の場合：`Ctrl + f/b` でカーソル前後移動、`Ctrl + n/p` で履歴の前後
- hjkl を覚えると vim 以外（man, less, screen 等）にも応用が効く

## 8. 行の中の移動

| キー | 動作 | 覚え方 |
|------|------|--------|
| `0` | 行の一番左（空白含む先頭） | 0番目の位置 |
| `^` | インデントを飛ばして最初の文字 | 正規表現の `^` に近い |
| `$` | 行の一番右（末尾） | 正規表現の `$` に近い |

### `0` と `^` の違い

```
    public static void main() {
↑ 0 はここ（空白の先頭）
    ↑ ^ はここ（インデント後の最初の文字）
```

- コードを書くときは `^` のほうが実用的（インデントの先頭に飛びたい場面が多い）
- `0` は行頭にインデントがないときや、行全体を選択したいときに使う

### 注意
- vim の `^` と正規表現の `^` は若干違う（正規表現は空白文字も含む）

## 9. 単語ジャンプ

| キー | 動作 | 覚え方 |
|------|------|--------|
| `w` | 次の単語の先頭へ | **w**ord |
| `b` | 前の単語の先頭へ | **b**ack |
| `e` | 次の単語の末尾へ | **e**nd |

### 大文字版（記号を無視して大きく飛ぶ）

| キー | 動作 |
|------|------|
| `W` | 記号を無視して次の単語の先頭へ（空白のみで止まる） |
| `B` | 記号を無視して前の単語の先頭へ |
| `E` | 記号を無視して次の単語の末尾へ |

### 使い分けの実例

```java
System.out.println("Hello");
```

- `w` → `System` → `.` → `out` → `.` → `println` → `(` → ...（記号で止まる）
- `W` → `System.out.println("Hello");` を一気に飛ばす（空白でのみ止まる）

Java のようにメソッドチェーンやドット記法が多い言語では `W`/`B`/`E` が実用的。

## 10. 文字へのジャンプ（f / t）

**vim ライフを変える最重要コマンドの1つ。** コピーや削除と組み合わせると真価を発揮する。

| キー | 動作 | 覚え方 |
|------|------|--------|
| `f{文字}` | その文字へカーソルを移動 | **f**ind / **f**ocus |
| `t{文字}` | その文字の**1つ手前**へカーソルを移動 | **t** = 手前 |
| `F{文字}` | その文字へカーソルを移動（**逆方向**） | 大文字 = 逆 |
| `T{文字}` | その文字の1つ手前へ（**逆方向**） | 大文字 = 逆 |

### 使用例

```java
function hoge(i, str) {
```

- 行頭で `f(` → `(` の位置に一発ジャンプ
- そこで `a` → 引数を追加できる

```java
function hoge(i, str) {
------------->
      f(
```

- 行末にいる状態で `F(` → 逆方向に `(` へジャンプ

### `f` と `t` の使い分け
- **`f`**：文字の上にカーソルを置きたいとき（挿入、ジャンプ目的）
- **`t`**：文字の手前まで範囲を指定したいとき（削除、コピーとの組み合わせで重要）

### 繰り返し（`;` と `,`）

同じ文字が複数ある場合、毎回 `f{文字}` と打つのは面倒。

| キー | 動作 | 覚え方 |
|------|------|--------|
| `;` | 同じ方向へ繰り返しジャンプ | `;` は点が2つ → 増やす方 |
| `,` | 逆方向へ繰り返しジャンプ | `,` は点が1つ → 減らす方 |

```
hoge hoge hoge hoge hoge
----->---->---->---->
  fh   ;    ;    ;
     <----<----<-----
       ,    ,    ,
```

- `fh` で最初の `h` に飛ぶ → `;` で次の `h` → `;` でさらに次
- 行き過ぎたら `,` で戻る

### 実践的な使い方のコツ
- その行に**1つしかない目立つ文字**を狙い撃つのがベスト
- `(`, `)`, `{`, `}`, `"`, `=` などの記号が狙い目
- 普通の文字（`a`, `e` 等）は複数回 `;` が必要になることが多い

## 11. 上下の高速移動

| キー | 動作 | 覚え方 |
|------|------|--------|
| `gg` | ファイルの先頭へ | |
| `G` | ファイルの末尾へ | **G**o（大文字=大げさに末尾まで） |
| `{数字}G` | 指定した行へジャンプ | 数字 + **G**o |
| `{数字}j` | 数字行分下へ | |
| `{数字}k` | 数字行分上へ | |
| `{` | 上の空行へジャンプ | |
| `}` | 下の空行へジャンプ | |

### 行ジャンプの実務的な使い方
- コンパイルエラー「Main.java:15: error:」→ `15G` で一発
- **行ジャンプは実務で死ぬほど使う**

### 空行ジャンプ `{` `}` の使い方
- メソッドとメソッドの間に空行がある場合、`}` で次のメソッドに飛べる
- コードリーディングで関数間を飛び回るときに便利

### 微調整のコツ
- 数字は `2`〜`4` が押しやすい
- `4j` → `4j` → `kk` のように大雑把に飛んで微調整
- 飛び過ぎを気にせずバシバシ行く。トータルのキータイプが減ればいい

## 12. 画面スクロール

| キー | 動作 | 覚え方 |
|------|------|--------|
| `Ctrl + f` | 画面1つ分先へ | **f**orward |
| `Ctrl + b` | 画面1つ分戻る | **b**ack |
| `Ctrl + d` | 画面半分先へ | **d**own |
| `Ctrl + u` | 画面半分戻る | **u**p |

### 移動の実践パターン（大→中→小）
1. `Ctrl + f` でバーンと大きく飛ぶ
2. 飛びすぎたら `Ctrl + u` で半画面戻す
3. `{数字}j` / `{数字}k` で行単位の微調整
4. `w` や `f` で行内の目的地へ

## 13. カーソル固定スクロール

カーソルはそのままで、**画面だけ動かす**。周辺のコードを確認したいときに使う。

| キー | 動作 |
|------|------|
| `Ctrl + y` | カーソル固定で上へスクロール（1行ずつ） |
| `Ctrl + e` | カーソル固定で下へスクロール（1行ずつ） |

### カーソル行を画面の特定位置にスクロール

| キー | 動作 | 覚え方 |
|------|------|--------|
| `zt` | カーソル行を画面最上部に | z + **t**op |
| `zz` | カーソル行を画面中央に | |
| `zb` | カーソル行を画面最下部に | z + **b**ottom |

### 実践的な使い方
```
Ctrl + f で飛びすぎた
→ Ctrl + u でちょいと戻す
→ 戻りすぎたので Ctrl + e でちょびちょび下げる
→ 狙ったところが真ん中に来たあたりで M（画面中央にカーソル）
```

## 15. 対応括弧ジャンプ

| キー | 動作 |
|------|------|
| `%` | 対応する括弧へ飛ぶ（`()`, `{}`, `[]`） |

### 使い方

```java
if (i == 10) {
    console.log(10);
}  ← % を押すと対応する { へ飛ぶ
```

- 対応括弧の上にカーソルを置いて `%` → 反対側の括弧に飛ぶ
- もう一度 `%` → 元に戻る

### 実務での使い方
- if 文の条件を確認したいとき：`$` で行末の `)` に飛ぶ → `%` で開始括弧へ
- インデントが崩れたコードで「どこがどうなってんだ？」→ `%` をポチポチ
- 同一行の括弧の中にカーソルがある場合、`%` で開始括弧に飛べる

### その他の括弧ジャンプ

| キー | 動作 |
|------|------|
| `[{` | カーソルを囲んでいる `{` へ飛ぶ |
| `]}` | カーソルを囲んでいる `}` へ飛ぶ |
| `[(` | カーソルを囲んでいる `(` へ飛ぶ |
| `])` | カーソルを囲んでいる `)` へ飛ぶ |

Java のように `{}` のネストが深いコードで「今自分がどのブロックの中にいるか」を確認するときに便利。

## 16. 削除

### 基本

| キー | 動作 | 覚え方 |
|------|------|--------|
| `x` | カーソル上の1文字を削除 | |
| `X` | カーソルの左の1文字を削除 | 大文字 = 逆方向 |
| `dd` | 1行削除 | **d**elete + **d**elete（行全体） |
| `D` | カーソルから行末まで削除 | `d$` と同じ |

### 移動との組み合わせ（`d` + 移動）

`d` は「削除」の**オペレータ**。移動コマンドと組み合わせる。

| キー | 動作 |
|------|------|
| `dw` | 次の単語まで削除 |
| `dW` | 記号無視で次の単語まで削除 |
| `db` | 前の単語まで削除 |
| `d$` | カーソルから行末まで削除 |
| `d0` | カーソルから行頭まで削除 |
| `d^` | カーソルからインデント後の先頭まで削除 |
| `dG` | カーソルからファイル末尾まで全削除 |
| `dgg` | カーソルからファイル先頭まで全削除 |
| `df{文字}` | 指定文字まで削除（文字含む） |
| `dt{文字}` | 指定文字の手前まで削除 |

### 数字との組み合わせ

| キー | 動作 |
|------|------|
| `3dd` | 3行削除 |
| `5x` | 5文字削除 |
| `d3w` | 3単語削除 |

### 重要な概念
- vim の削除は実際には「カット」。削除した内容は**レジスタ（クリップボード）に保存**される
- `dd` で行を削除 → `p` で貼り付け → 行の移動になる

## 17. コピー（ヤンク）& ペースト

vim では「コピー」を「**ヤンク（yank）**」と呼ぶ。

### 基本

| キー | 動作 | 覚え方 |
|------|------|--------|
| `yy` | 1行コピー | **y**ank + **y**ank |
| `Y` | 1行コピー（`yy` と同じ） | |
| `p` | カーソルの**後ろ**にペースト | **p**aste / **p**ut |
| `P` | カーソルの**前**にペースト | 大文字 = 逆方向 |

### 移動との組み合わせ（`y` + 移動）

| キー | 動作 |
|------|------|
| `yw` | 次の単語までコピー |
| `yW` | 記号無視で次の単語までコピー |
| `y$` | カーソルから行末までコピー |
| `y0` | カーソルから行頭までコピー |
| `yf{文字}` | 指定文字までコピー |
| `yt{文字}` | 指定文字の手前までコピー |
| `3yy` | 3行コピー |

### `p` と `P` の挙動
- **行単位のヤンク**（`yy` 等）→ `p` で下の行に、`P` で上の行にペースト
- **文字単位のヤンク**（`yw` 等）→ `p` でカーソルの右に、`P` でカーソルの左にペースト

## 18. 置換・変更

### 1文字置換

| キー | 動作 | 覚え方 |
|------|------|--------|
| `r{文字}` | カーソル上の文字を1文字だけ置き換え | **r**eplace |
| `R` | 上書きモード（連続して置き換え） | 大文字 = 大げさ |

### 変更（削除 + 入力モード）

`c` は「変更」の**オペレータ**。指定範囲を削除して入力モードに入る。

| キー | 動作 |
|------|------|
| `cw` | 単語を変更（削除して入力モード） |
| `cW` | 記号無視で単語を変更 |
| `cc` | 行全体を変更 |
| `C` | カーソルから行末まで変更（`c$` と同じ） |
| `cf{文字}` | 指定文字まで変更 |
| `ct{文字}` | 指定文字の手前まで変更 |

### `d` と `c` の違い
- `d` → 削除してノーマルモードのまま
- `c` → 削除して**入力モードに入る**（削除した箇所をすぐ書き直す）

## 19. 検索

| キー | 動作 | 覚え方 |
|------|------|--------|
| `/{検索語}` + `Enter` | 下方向に検索 | |
| `?{検索語}` + `Enter` | 上方向に検索 | |
| `n` | 次の検索結果へ | **n**ext |
| `N` | 前の検索結果へ | 大文字 = 逆方向 |
| `*` | カーソル上の単語を下方向に検索 | |
| `#` | カーソル上の単語を上方向に検索 | |

### 検索のコツ
- `/` で検索 → `n` で次々飛ぶ → 目的の場所で編集、が基本パターン
- `*` はカーソルを合わせるだけで検索できるので、変数の使用箇所を確認するときに非常に便利
- 検索ハイライトを消すには `:noh`（`:nohlsearch` の省略）

## 20. 置換コマンド

コマンドモードで実行する。

| `:s/old/new/g` | 現在行の全ての `old` を `new` に置換 |
| `:%s/old/new/g` | ファイル全体で置換 |

### 実務での使い方

## 22. ビジュアルモード（範囲選択）

テキストを視覚的に選択してから操作する。

| キー | 動作 |
|------|------|
| `v` | 文字単位の選択開始 |
| `V` | 行単位の選択開始（大文字 = 大げさ） |
| `Ctrl + v` | 矩形（ブロック）選択開始 |

### 選択してからの操作
1. `v` または `V` で選択開始
2. 移動コマンド（`j`, `w`, `$` 等）で範囲を広げる
3. 操作を実行：
   - `d` → 選択範囲を削除
   - `y` → 選択範囲をコピー
   - `c` → 選択範囲を変更
   - `>` → 選択範囲をインデント
   - `<` → 選択範囲をインデント解除

### 矩形選択（`Ctrl + v`）
- 複数行の同じ位置を一括で編集できる
- 例：複数行の先頭にコメント `//` を一括挿入

## 24. 繰り返し（ドットコマンド）

| キー | 動作 |
|------|------|
| `.` | **直前の変更操作を繰り返す** |

vim の中でも**最も強力なコマンドの1つ**。

### 使用例
1. `cw` で単語を変更 → `new_word` と入力 → `Ctrl + C`
2. 次の同じ単語に移動（`n` や `w` で）
3. `.` を押す → 同じ変更が繰り返される

「検索（`/` or `*`）→ 変更 → `n` で次へ → `.` で繰り返し」のパターンは日常的に使う。

## 25. テキストオブジェクト

vim の最も強力な概念の1つ。「**括弧の中身**」「**単語全体**」などを論理的な単位で操作できる。

### 基本形
- `i` = **i**nner（内側、区切り文字を含まない）
- `a` = **a**round（外側、区切り文字を含む）

### 操作 + テキストオブジェクト

| キー | 動作 |
|------|------|
| `diw` | 単語を削除（**d**elete **i**nner **w**ord） |
| `daw` | 単語と周辺の空白も削除 |
| `di(` / `dib` | `()` の中身を削除 |
| `da(` / `dab` | `()` ごと削除 |
| `di{` / `diB` | `{}` の中身を削除 |
| `da{` / `daB` | `{}` ごと削除 |
| `di"` | `""` の中身を削除 |
| `da"` | `""` ごと削除 |
| `di'` | `''` の中身を削除 |
| `ci(` | `()` の中身を変更（削除して入力モード） |
| `ci"` | `""` の中身を変更 |
| `yi(` | `()` の中身をコピー |
| `ya{` | `{}` ごとコピー |

### Java での実用例

```java
System.out.println("Hello World");
```

- カーソルが `"Hello World"` の中にある状態で `ci"` → `"` の中身が消えて入力モード → 新しい文字列を入力

```java
if (x > 0 && y < 10) {
```

- カーソルが `()` の中にある状態で `di(` → 条件部分だけが消える

### なぜ強力なのか
- カーソルが括弧やクォートの**中のどこにあっても**動作する
- わざわざ先頭に移動する必要がない

## 27. .vimrc の基本設定

ホームディレクトリに `~/.vimrc` を作成し、以下を書いておくと便利。
コマンドモードの `:` は不要で、直接書く。

```vim
" 行番号を表示
set number

" 相対行番号を表示（数字 + j/k が使いやすくなる）
set relativenumber

" タブ幅を4スペースに（Java 標準）
set tabstop=4
set shiftwidth=4
set expandtab

" 検索でハイライト
set hlsearch

" インクリメンタルサーチ（入力しながら検索）
set incsearch

" 検索で大文字小文字を無視（大文字を含む場合は区別）
set ignorecase
set smartcase

" 対応括弧を表示
set showmatch

" シンタックスハイライト有効
syntax on

" カーソル行をハイライト
set cursorline

" ステータスラインを常に表示
set laststatus=2

" エンコーディング
set encoding=utf-8

" バックスペースの挙動を普通にする
set backspace=indent,eol,start
```

### 設定の省略形

| 正式名 | 省略形 |
|--------|--------|
| `tabstop` | `ts` |
| `shiftwidth` | `sw` |
| `expandtab` | `et` |
| `number` | `nu` |
| `relativenumber` | `rnu` |
| `hlsearch` | `hls` |
| `incsearch` | `is` |

## 29. vim の移動体系まとめ図

```
【ファイル全体】
  gg .......................... G
  (先頭)                    (末尾)
          15G (15行目へ)

【画面単位スクロール】
  Ctrl+b ← 1画面 → Ctrl+f
  Ctrl+u ← 半画面 → Ctrl+d

【画面内ジャンプ】
  H (上) ── M (中) ── L (下)

【カーソル固定スクロール】
  Ctrl+y (上へ)    Ctrl+e (下へ)
  zt (先頭へ)  zz (中央へ)  zb (末尾へ)

【行間移動】
  k (上) / j (下) / {数字}j / {数字}k
  { (上の空行) / } (下の空行)

【行内移動】
  0 ──── ^ ──── w/W ──── e/E ──── $ 
  (行頭)  (indent後)  (単語)  (単語末)  (行末)
          b/B ←── (戻る)
  
  f{char} → 文字へ   t{char} → 手前へ
  F{char} ← 逆方向   T{char} ← 逆方向
  ; → 繰り返し        , ← 逆繰り返し

【対応括弧】
  % (対応括弧へトグル)
```

## オペレータと移動の組み合わせ早見表

vim のオペレータ（`d`, `c`, `y`）は全て同じ移動コマンドと組み合わせられる。

| | `d`（削除） | `c`（変更） | `y`（コピー） |
|---|---|---|---|
| `w` | `dw`（単語削除） | `cw`（単語変更） | `yw`（単語コピー） |
| `$` | `d$`（行末まで削除） | `c$`（行末まで変更） | `y$`（行末までコピー） |
| `f{char}` | `df{char}` | `cf{char}` | `yf{char}` |
| `t{char}` | `dt{char}` | `ct{char}` | `yt{char}` |
| `i(` | `di(`（()内削除） | `ci(`（()内変更） | `yi(`（()内コピー） |
| `i"` | `di"`（""内削除） | `ci"`（""内変更） | `yi"`（""内コピー） |
| `G` | `dG`（末尾まで削除） | `cG`（末尾まで変更） | `yG`（末尾までコピー） |

**パターン：`オペレータ` + `移動/テキストオブジェクト` = その範囲に操作を適用**

この組み合わせを理解すれば、個別のコマンドを暗記する量が激減する。
