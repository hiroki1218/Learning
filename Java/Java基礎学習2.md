**Publicクラスとファイル名のルールに関して**
Publicクラスと.javaファイル名は **同一** である必要がある。
まずPublicクラスとは外部から参照できるクラスを指します。
外部から参照して使用する際にjavacコンパイラは.javaファイル名と一致するクラス名を探索します。そのため名称が一致しないと見つからずコンパイルエラーが発生します。なのでPublicクラスを使用する際はファイル名と同一である必要性をルールとして強制させている。

## mainメソッドの役割と各キーワードの意味(public,static,void,String[]args)

| キーワード | 役割 |
| --- | --- |
| main | JVMがプログラム実行時に最初に呼び出しするエントリーポイント |
| public | 外部から参照可能なメソッドであることの宣言 |
| static | インスタンスを生成しなくてもメソッドが使用可能になる|
| void | メソッドの返値が無いこと宣言 |
| String[]args | コマンドライン引数と言われており、プログラム実行時に引数として値を渡すことが出来る |

## ArrayIndexOutOfBoundsExceptionとは
javaの例外の1つで配列に存在しないインデックスにアクセスしようとしたときに
発生する実行時エラーです。
今回はコマンドライン引数を試す際に渡す値の記述を忘れており
何もない配列を表示しようとしてエラーが発生。
そのため静的型付け言語であるJavaにもコンパイル時に検出出来ないエラーがあります。
