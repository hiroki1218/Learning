# Java基礎学習1

## Javaの特徴

- .javaファイルをバイトコードに変換し、JVM上で実行するためOS毎の差異を吸収する。
- 静的型付け言語
- オブジェクト指向
- ガベージコレクション

## コンパイルから実行までの流れ

Javaソースコードの記述　→ **javac** コマンドにてコンパイル → バイトコードに変換 → バイトコードをJVMで読み取る　→ 処理を実行

## コンパイルの種類

| 名称                               | 特徴                                                                                     | メリット                         | デメリット                                           |
| ---------------------------------- | ---------------------------------------------------------------------------------------- | -------------------------------- | ---------------------------------------------------- |
| 事前コンパイル(AOT:Ahead-Of-Time)  | プログラム実行する前にソースコードを一括で機械語に変換してから実行可能ファイルを生成する | 実行速度が速く、コンパイラが不要 | コンパイルに時間がかかり環境ごとに再コンパイルが必要 |
| 実行時コンパイル(JIT:Just-In-Time) | プログラム実行時に必要な箇所のみをJITコンパイラが機械語にコンパイルを行う         | 実行環境に合わせて最適化が可能   | 初回起動時にコンパイラのオーバーヘッドが発生する     |

**用語の補足**

- コンパイル：ソースコードをコンピューターが理解できるように機械語に変換すること
- コンパイラ：プログラムを **実行する前** に　**まとめて機械語** に変換する仕組み
- インタプリタ：プログラムを **1行ずつ** しながら実行する仕組み
- オーバーヘッド：**本来行いたい処理以外(時間やメモリ)にかかるコスト**　のこと
- ガベージコレクション：使用されていない不要なメモリを削除して領域を確保する仕組み

## JVMの実行の仕組み

まずJavaは **コンパイラ** / **インタプリタ** を両方使用する言語である。
ソースコードを **`javac`**コマンドにて中間コードである**バイトコード**に変換する。
バイトコードを**JVM**が**インタプリタ**のように実行します。
さらにJVMの中のJITコンパイラがよく使用されている箇所を機械語にコンパイルし高速化している仕組みとなっている。
